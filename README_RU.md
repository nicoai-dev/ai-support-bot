Production-ready ИИ-ассистент для Telegram на архитектуре **RAG (Retrieval-Augmented Generation)**. Полностью локальная работа через **Ollama** — без облачных API, без абонентских расходов, с полной конфиденциальностью данных.

> **Коротко:** Telegram-бот, который отвечает на вопросы клиентов о магазине с помощью локальной LLM + векторного поиска по базе знаний. Стриминг ответов, многоуровневая защита от галлюцинаций, встроенный Mini App-магазин и оркестратор, который управляет ботом, веб-сервером и SSH-туннелем автоматически.

## ✨ Ключевые возможности

### 🧠 RAG-пайплайн
- **Семантический поиск** через `SentenceTransformers` (`paraphrase-multilingual-MiniLM-L12-v2`) + `ChromaDB` с косинусной метрикой и нормализованными эмбеддингами.
- **Чанкинг** через `langchain-text-splitters` с `RecursiveCharacterTextSplitter` (800 символов, перекрытие 150) и умными разделителями (`===`, `---`, `\n\n` и др.), которые уважают структуру документов.
- **Контекстный поиск**: бот анализирует историю диалога для обработки уточняющих вопросов — пользователю не нужно повторяться.
- **Фильтрация по дистанции**: чанки с косинусным расстоянием > 0.9 молча отбрасываются, не давая шуму попасть в контекст LLM.

### 🛡️ Система защиты от галлюцинаций (многоуровневая)
1. **Системный промпт**: 6 жёстких правил, зашитых в промпт LLM — ответы строго по контексту, запрет на выдуманные товары/URL/контакты, естественный тон общения, ответы только на русском, категориальные ответы на общие вопросы.
2. **Программные гардрейлы** (`rag/guardrails.py`): regex-валидация каждого ответа перед отправкой:
   - Обнаружение выдуманных телефонов, email и URL, не совпадающих с белым списком.
   - Детекция маркеров утечки промпта ("согласно инструкции", "в моей базе").
   - Если срабатывает ≥ 3 предупреждений одновременно, весь ответ заменяется безопасным фолбэком с перенаправлением к живому менеджеру.
3. **Мягкий фолбэк**: если ни один чанк не прошёл порог дистанции, бот пропускает LLM и сразу выдаёт заготовленный ответ с реальными контактами поддержки.

### ⚡ Потоковые ответы (Streaming)
- Бот использует **Streaming Chat API** Ollama — токены появляются в Telegram-сообщении в реальном времени.
- Троттлинг обновлений (каждые 1.5 сек) для соблюдения лимитов Telegram API, с тихим восстановлением при флуд-ошибках.
- Семафор (`MAX_CONCURRENT_GENERATIONS = 1`) сериализует запросы генерации для предотвращения конкуренции за ресурсы на ограниченном железе.

### 🛒 Telegram Mini App (WebApp)
- Полностью адаптивный каталог-витрина, встроенный как **Telegram Mini App** (WebApp).
- Живой поиск товаров, вкладки категорий (Устройства, Софт, Услуги), модальные окна товаров с glassmorphism-дизайном.
- Интерактивная корзина с управлением количеством, итогами в реальном времени и оформлением заказа, который отправляет структурированные данные обратно в бот.
- Аватар и имя пользователя передаются через URL-параметры и отображаются в шапке.
- Каталог товаров загружается из локального `products.json` (22K+ записей).

### 🔄 Оркестратор (`run_all.py`)
Единый скрипт-супервизор, управляющий всем стеком:
1. **Убивает «осиротевшие» процессы** — чистит зависшие экземпляры бота и SSH-туннелей от предыдущих запусков (Windows-специфичная логика через `Get-CimInstance`).
2. **Запускает HTTP-сервер** (порт 8000) для раздачи WebApp из папки `webapp/`.
3. **Устанавливает SSH-туннель** к `localhost.run` с keep-alive (интервал 15 сек, макс. 3 промаха).
4. **Автоматически обновляет `.env`** новым URL туннеля, чтобы бот всегда имел актуальный `WEBAPP_URL`.
5. **Мониторит оба процесса** в цикле:
   - Авто-рестарт бота при падении.
   - **Активная проверка здоровья туннеля**: пингует URL туннеля и обнаруживает «мёртвые» туннели (страницы "no tunnel here", HTTP 502/503/504) — даже если сам SSH-процесс ещё жив.
   - Реконнект с экспоненциальным бэкоффом (до 5 попыток, макс. задержка 30 сек).
6. **Graceful shutdown** по `Ctrl+C` — завершает все дочерние процессы с таймаутами.

### 🏗️ Production-закалка
- **Глобальный `ErrorHandlingMiddleware`**: ловит любое необработанное исключение в любом хендлере и отправляет вежливые извинения пользователю.
- **Фоновая очистка памяти**: корутина запускается каждые 5 минут, удаляя истёкшие сессии (TTL: 10 мин) и устаревшие записи rate-limit (>1 часа).
- **Rate limiting**: 3-секундный кулдаун на пользователя через модуль памяти.
- **Валидация длины сообщений**: отклоняет сообщения длиннее 1000 символов.
- **Windows `wakeup_loop`**: async sleep 0.5 сек для поддержания отзывчивости event loop на `Ctrl+C` в Windows (известная особенность `asyncio`).
- **Singleton `aiohttp`-сессия** с `asyncio.Lock` для защиты от race condition при конкурентном доступе.
- **Последовательность graceful shutdown**: отмена фоновых задач → ожидание с таймаутом → закрытие aiohttp-сессии → закрытие сессии бота.

│   ├── prompts/             # Системный промпт (system.txt)
│   └── memory.db            # SQLite-хранилище памяти
├── webapp/
│   ├── index.html           # Фронтенд Mini App
│   ├── style.css            # UI-стили с glassmorphism
│   ├── app.js               # Каталог товаров, логика корзины, интеграция Telegram WebApp API
│   ├── products.json        # База товаров
│   └── avatars/             # Ассеты аватаров
├── tests/
│   └── test_guardrails.py   # Юнит-тесты валидации гардрейлов
├── Dockerfile               # Образ на базе Python 3.10-slim
├── docker-compose.yml       # Конфигурация сервиса с host networking для доступа к Ollama

## 📦 Быстрый старт

### Предварительные требования
- **Python 3.10+** (для локальной разработки)
- **[Ollama](https://ollama.ai/)** установлена и запущена
- **Docker и Docker Compose** (для контейнеризованного деплоя)
- **SSH-клиент** (для туннеля в режиме `run_all.py`)

### Вариант 1: Локальная разработка (Windows)

```bash
# 1. Клонируем репозиторий
git clone https://github.com/your-username/ai-support-bot.git
cd ai-support-bot

# 2. Создаём и активируем виртуальное окружение
python -m venv venv
.\venv\Scripts\activate

# 3. Устанавливаем зависимости
pip install -r requirements.txt

# 4. Настраиваем окружение
copy .env.example .env
# Редактируем .env — вставляем BOT_TOKEN из @BotFather
```

**Запуск только бота** (без WebApp, без туннеля):
```bash
python main.py
# или
run_bot.bat
```

**Запуск полного стека** (бот + сервер WebApp + SSH-туннель):
```bash
python run_all.py
# или
run_all.bat
```

### Вариант 2: Docker

```bash
# 1. Настраиваем .env
copy .env.example .env
# Редактируем .env — указываем BOT_TOKEN, при необходимости меняем OLLAMA_BASE_URL

# 2. Добавляем файлы базы знаний
# Помещаем .txt файлы в data/knowledge_base/

# 3. Запускаем
docker-compose up --build -d
```

> **Примечание:** Docker-контейнер использует `network_mode: "host"`, чтобы бот мог достучаться до Ollama, запущенной на localhost. Если Ollama на другом хосте, обновите `OLLAMA_BASE_URL` в `.env`.

### Переменные окружения

| Переменная | По умолчанию | Описание |
|------------|-------------|----------|
| `BOT_TOKEN` | — (обязательно) | Токен Telegram-бота от [@BotFather](https://t.me/BotFather) |
| `OLLAMA_BASE_URL` | `http://localhost:11434` | Эндпоинт Ollama API |
| `OLLAMA_MODEL` | `qwen2.5-coder:32b` | Название модели LLM |
| `WEBAPP_URL` | `https://nico-market-catalog.loca.lt` | Публичный URL WebApp (автоматически обновляется `run_all.py`) |

## 📝 База знаний

Знания бота берутся из обычных `.txt` файлов в `data/knowledge_base/`. При старте они:
1. Загружаются и разбиваются на чанки (800 символов, перекрытие 150).
2. Векторизуются моделью `paraphrase-multilingual-MiniLM-L12-v2`.
3. Сохраняются в ChromaDB с метаданными об источнике.

Текущая структура базы знаний:
| Файл | Содержание |
|------|-----------|
| `01_about.txt` | Информация о компании и история |
| `02_contacts.txt` | Официальные контакты и часы работы |
| `03_catalog.txt` | Полный каталог товаров с ценами |
| `04_policies.txt` | Политики возврата, доставки и гарантии |
| `05_subscriptions.txt` | Подписочные планы |
| `06_faq.txt` | Часто задаваемые вопросы |
| `07_team.txt` | Команда |
| `08_bot_persona.txt` | Личность и правила поведения бота |

Для обновления: отредактируйте/добавьте `.txt` файлы и перезапустите бота — индекс пересоберётся автоматически.

## ⚙️ Конфигурационные константы

| Константа | Значение | Расположение |
|-----------|---------|--------------|
| `MEMORY_MAX_MESSAGES` | 4 | `config.py` — макс. сообщений в сессии |
| `MEMORY_SESSION_TTL` | 600 сек (10 мин) | `config.py` — время жизни сессии |
| `OLLAMA_TIMEOUT` | 180 сек | `config.py` — таймаут генерации LLM |
| `RATE_LIMIT` | 3.0 сек | `bot/handlers.py` — кулдаун на пользователя |
| `MAX_CONCURRENT_GENERATIONS` | 1 | `bot/handlers.py` — семафор генерации |
| `distance_threshold` | 0.9 | `rag/retriever.py` — макс. косинусная дистанция для чанков |
| `chunk_size` / `chunk_overlap` | 800 / 150 | `rag/loader.py` — параметры разбивки текста |
| `temperature` | 0.5 | `rag/chain.py` — температура сэмплирования LLM |
| `num_predict` | 512 | `rag/chain.py` — макс. генерируемых токенов |
