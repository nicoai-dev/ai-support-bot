## Часть 1: Анализ текущего состояния по скриншоту

### Что работает хорошо

1. **Память работает** — бот помнит номер заказа 318 и контекст об отмене через 3 минуты молчания
2. **Тон правильный** — деловой, от первого лица, без "воды"
3. **Переход на менеджера** — корректно направляет на службу поддержки
4. **Нет повторных приветствий** — промпт работает

### Что нужно улучшить (видно из скриншота)

#### 🔴 Проблема 1: Бот "выдумывает" процессы, которых нет в базе знаний

> «Я передам его в службу поддержки клиентов для обработки вашей просьбы о отмене. Обычно на рассмотрение запроса уходит до 24 часов.»

В `rules.txt` **нет** информации о:
- "службе поддержки клиентов" как отдельном отделе
- "рассмотрении запроса до 24 часов"
- "менеджер свяжется с вами"

Бот **галлюцинирует** — выдумывает бизнес-процессы, которых нет в контексте. Это самая критическая проблема, т.к. клиент ожидает звонок менеджера, которого не будет.

**Причина:** В `rules.txt` сказано только:
> «Заказ можно отменить только до момента передачи его курьеру. После выезда курьера отмена невозможна.»

Бот должен спросить: "Заказ уже передан курьеру?" и дать чёткий ответ по правилам, а не выдумывать "службу обработки".

#### 🟡 Проблема 2: Избыточность ответов

Последнее сообщение на скриншоте — бот повторяет **весь** предыдущий ответ слово в слово на вопрос «Вы ещё тут?». Промпт запрещает повторять, но 32B модель всё равно делает это, потому что в истории хранится полный текст прошлого ответа и модель его "пересказывает".

#### 🟡 Проблема 3: Бот принимает "заказы на отмену" без проверки условий

По правилам бот **обязан** сначала проверить: курьер уже выехал или нет? Вместо этого он сразу говорит "я передам запрос". Бот должен задать уточняющий вопрос.

## Часть 2: Проблемы в коде

### 🔴 Критические

#### 1. Нет защиты от галлюцинаций (anti-hallucination)

Промпт не содержит **жёсткого запрета** на выдумывание информации, которой нет в контексте. Текущий промпт говорит только "используй контекст", но не говорит "ЗАПРЕЩЕНО выдумывать факты".

**Решение — добавить в `SYSTEM_PROMPT`:**
```python
ANTI_HALLUCINATION = """
КРИТИЧЕСКИ ВАЖНО:
- ЗАПРЕЩЕНО выдумывать сроки, процессы, отделы или процедуры, которых НЕТ в контексте.
- Если в контексте нет ответа на вопрос — скажи: "К сожалению, у меня нет точной информации по этому вопросу. 
  Рекомендую уточнить у менеджера по телефону +7 999 000-00-00."
- НЕ говори "я передам", "я проверю", "я свяжусь" — ты не имеешь доступа к системам магазина.
- Ты можешь ТОЛЬКО: отвечать на вопросы по базе знаний и направлять к менеджеру.
"""
```

#### 2. `aiohttp.ClientSession()` создаётся на каждый запрос

В `chain.py:53` — `async with aiohttp.ClientSession()` создаёт новый TCP-коннект на каждое сообщение. Это медленно и тратит ресурсы.

**Решение — singleton сессия:**
```python
_session = None

async def get_session():
    global _session
    if _session is None or _session.closed:
        timeout = aiohttp.ClientTimeout(total=120)
        _session = aiohttp.ClientSession(timeout=timeout)
    return _session
```

#### 3. `timeout=60` — недостаточно для 32B модели

В `chain.py:65` — таймаут 60 секунд. Qwen 32B на CPU может генерировать ответ **до 2-3 минут**. Если модель думает дольше 60 сек, клиент получит ошибку.

**Решение:** Увеличить до 180 сек или вынести в `config.py`:
```python
OLLAMA_TIMEOUT = int(os.getenv("OLLAMA_TIMEOUT", "180"))
```

#### 4. `timeout=60` передан как `int`, а не `aiohttp.ClientTimeout`

`aiohttp` ожидает объект `ClientTimeout`, а не число. Текущий код передаёт `timeout=60` — это `timeout` для `session.post()`, который aiohttp интерпретирует как `total`, но это **deprecated** поведение и может сломаться.

**Решение:**
```python
timeout = aiohttp.ClientTimeout(total=180)
```

## Часть 3: Улучшения промпта

### Текущая проблема: бот "обещает" действия, которые не может выполнить

Бот говорит «Я передам», «Я проверю» — это создаёт ложные ожидания. В реальности бот не интегрирован ни с какой CRM или системой заказов.

### Предлагаемый улучшенный промпт

```python
SYSTEM_PROMPT = """Ты — Нико, сотрудник поддержки Nico Market (мужской род).

РОЛЬ:
- Ты консультант. Ты отвечаешь на вопросы по правилам магазина.
- Ты НЕ имеешь доступа к системе заказов, складу или CRM.
- Ты НЕ можешь "передать", "проверить", "оформить" или "обработать" что-либо.

ПРАВИЛА ОТВЕТОВ:
1. Отвечай ТОЛЬКО на основе предоставленного Контекста.

- НЕ пересказывай вопрос клиента.
- НЕ упоминай "базу знаний", "контекст", "инструкции".
- НЕ говори "я передам/проверю/свяжусь" — у тебя нет такой возможности.
```

## Часть 4: Архитектурные улучшения (следующий уровень)

### 🔴 Приоритет 1: Streaming-ответы

32B модель генерирует ответ ~30-120 секунд. Клиент сидит и ждёт, глядя на "🔄 Обрабатываю...". Streaming решает это — ответ появляется постепенно.

```python
async def generate_answer_stream(question, context_chunks, chat_history=None):
    """Потоковая генерация — yield по частям."""
    # ... формируем messages как раньше ...
    
    session = await get_session()
    async with session.post(
        f"{OLLAMA_BASE_URL}/api/chat",
        json={"model": OLLAMA_MODEL, "messages": messages, "stream": True, ...}
    ) as response:
        buffer = ""
        async for line in response.content:
            data = json.loads(line)
            token = data.get("message", {}).get("content", "")
            buffer += token
            yield buffer  # текущий буфер
```

В handler — обновлять сообщение каждые ~1-2 секунды:
```python
import asyncio

full_text = ""
last_edit = 0
async for partial in generate_answer_stream(question, chunks, chat_history):
    full_text = partial
    now = time.time()
    if now - last_edit > 1.5:  # Telegram rate limit ~1 edit/sec
        await processing_msg.edit_text(partial + " ▌")
        last_edit = now

await processing_msg.edit_text(full_text)
```

### 🟡 Приоритет 2: Логирование запросов для аналитики

Сохранять каждый диалог для анализа: какие вопросы задают, где бот ошибается.

```python
# bot/logger.py
import json, os
from datetime import datetime

LOG_DIR = os.path.join(os.path.dirname(__file__), "..", "data", "logs")
os.makedirs(LOG_DIR, exist_ok=True)

def log_interaction(user_id: int, question: str, answer: str, chunks: list[str]):
    entry = {
        "timestamp": datetime.now().isoformat(),
        "user_id": user_id,
        "question": question,
        "answer": answer,
        "chunks_used": len(chunks),
    }
    filepath = os.path.join(LOG_DIR, f"{datetime.now().strftime('%Y-%m-%d')}.jsonl")
    with open(filepath, "a", encoding="utf-8") as f:
        f.write(json.dumps(entry, ensure_ascii=False) + "\n")
```
